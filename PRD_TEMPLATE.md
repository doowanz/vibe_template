# PRD.md — 제품 요구사항 문서

> **PRD란?** Product Requirements Document의 줄임말입니다.
> "내가 만들고 싶은 것"을 AI가 이해할 수 있게 정리한 문서예요.
>
> Claude에게 PRD를 주면, 어떻게 만들지 스스로 계획하고 코드를 짜줍니다.
> 막연하게 "만들어줘" 하는 것보다 훨씬 원하는 결과가 나옵니다.

---

## PRD를 쓰는 이유

| 방식 | 결과 |
|---|---|
| "맛집 사이트 만들어줘" | Claude가 마음대로 해석해서 만듦 |
| PRD를 주고 "이 문서대로 만들어줘" | 내가 원하는 대로 만들어줌 |

---

## PRD 구성 요소

| 항목 | 설명 |
|---|---|
| 서비스 개요 | 한 줄로 설명하는 "이게 뭐야?" |
| 목표 사용자 | 누가 쓸 것인가 |
| 핵심 기능 | 반드시 있어야 하는 기능 목록 |
| 화면 구성 | 어떤 페이지가 필요한가 |
| 데이터 구조 | 어떤 정보를 저장할 것인가 |
| 제외 범위 | 이번엔 만들지 않을 것 |

## 샘플 PRD — 팀 점심 메뉴 투표 서비스

```
# 서비스 개요
팀원들이 점심 메뉴를 투표로 정하는 웹서비스.
"오늘 뭐 먹지?" 고민을 없애줍니다.

# 목표 사용자
- 같이 점심을 먹는 소규모 팀 (5~20명)
- 매일 메뉴 결정에 시간을 낭비하는 직장인

# 핵심 기능

## 반드시 있어야 하는 것 (Must Have)
- 메뉴 등록: 누구나 메뉴 이름을 추가할 수 있다
- 투표: 메뉴마다 추천 버튼(👍)을 누를 수 있다
- 결과 확인: 추천 수가 많은 순서대로 정렬된다
- 오늘의 메뉴: 추천 수 1위 메뉴가 크게 표시된다

## 있으면 좋은 것 (Nice to Have)
- 하루가 지나면 투표가 자동 초기화된다
- 메뉴에 이미지를 추가할 수 있다

## 이번엔 만들지 않을 것 (Out of Scope)
- 로그인/회원가입 (누구나 익명으로 투표 가능)
- 식당 위치나 지도 연동
- 알림 기능

# 화면 구성

## 메인 페이지 (/)
- 상단: 서비스 제목 + 오늘 날짜
- 중앙: 추천 수 1위 메뉴를 크게 표시 (오늘의 메뉴)
- 하단: 전체 메뉴 목록 + 각 메뉴의 추천 버튼과 추천 수

## 메뉴 추가
- 메인 페이지 상단에 입력창 + 추가 버튼
- 이름만 입력하면 등록 완료

# 데이터 구조 (Supabase 테이블)

## menus 테이블
| 컬럼명 | 타입 | 설명 |
|---|---|---|
| id | uuid | 고유 식별자 (자동 생성) |
| name | text | 메뉴 이름 (예: 김치찌개) |
| vote_count | integer | 추천 수 (기본값: 0) |
| created_at | timestamp | 등록 시각 |

# 디자인 방향
- 밝고 식욕을 돋우는 색상 (주황, 노랑 계열)
- 모바일에서도 잘 보여야 함 (반응형)
- shadcn/ui 컴포넌트 사용

# Claude에게 전달할 프롬프트
@PRD_TEMPLATE.md 를 읽고 이 서비스를 만들어줘.
Next.js + Tailwind CSS + Supabase 스택으로 구현해줘.
CLAUDE.md 규칙도 함께 지켜줘.
```

## Claude에게 PRD 작성을 맡기는 방법

PRD 작성이 어렵다면 Claude에게 먼저 도움을 요청하세요.

```
나는 [이런 서비스]를 만들고 싶어.
PRD.md 템플릿에 맞게 PRD를 작성해줘.
내가 확인하고 수정할게.
```
