# 서비스 개요
이 서비스는 서울 삼성동에서 근무하는 팀을 위해, 실시간 날씨를 자동으로 확인하고
그 날씨에 어울리는 점심 메뉴 3가지를 AI가 이유와 함께 추천해주는 서비스입니다.
"오늘 뭐 먹지?" + "날씨가 이런데 뭐가 어울리지?" 두 가지 고민을 한 번에 해결합니다.

# 목표 사용자
- 서울 삼성동(코엑스 인근) 사무실에서 근무하는 소규모 팀
- 매일 점심시간마다 메뉴 고민을 반복하는 직장인

# 핵심 기능

## 반드시 있어야 하는 것 (Must Have)
- **위치 고정**: 위치는 항상 "서울 삼성동"으로 고정한다 (사용자가 위치를 입력/선택할 필요 없음)
- **실시간 날씨 조회**: 날씨 API를 호출해서 현재 기온, 체감온도, 날씨 상태(맑음/흐림/비/눈 등)를 가져온다
- **날씨 정보 표시**: 가져온 날씨 정보를 화면 상단에 보기 쉽게 보여준다
- **AI 메뉴 추천**: 날씨 정보를 Claude API에 전달해서, 그 날씨에 어울리는 점심 메뉴 3개를 "추천 이유"와 함께 받아온다
  - 예: "비 오는 날엔 뜨끈한 국물이 생각나죠 → 순댓국"
- **추천 결과 표시**: 메뉴 3개를 카드 형태로 나란히 보여준다 (메뉴명 + 한 줄 추천 이유)
- **다시 추천받기**: 버튼을 누르면 같은 날씨 기준으로 메뉴 추천을 다시 요청할 수 있다
- **로딩/에러 처리**: 날씨 조회 중, AI 추천 생성 중 상태를 표시하고, API 실패 시 안내 문구와 재시도 버튼을 보여준다

## 있으면 좋은 것 (Nice to Have)
- 같은 날씨 조건이면 일정 시간(예: 30분) 동안 추천 결과를 캐싱해서 API 호출 비용 절약
- 오늘 추천받았던 메뉴 히스토리 보기
- 날씨 상태에 따라 배경 색상/분위기가 바뀌는 연출 (맑음: 밝은 하늘색, 비: 차분한 블루그레이 등)
- 메뉴 카드에 어울리는 이모지 표시

## 이번엔 만들지 않을 것 (Out of Scope — v2로 보류)
- **팀 투표 기능** (추천된 메뉴 중 팀원들이 투표해서 최종 메뉴를 정하는 기능 → v2)
- 회원가입/로그인
- 위치 변경 기능 (다른 지역 날씨 조회)
- 메뉴 즐겨찾기/저장
- 알림 기능

# 화면 구성

## 메인 페이지 (`/`)
- **상단**: 서비스 제목 + 오늘 날짜
- **날씨 카드**: 삼성동 현재 날씨 아이콘, 기온(°C), 체감온도, 날씨 상태 텍스트 (예: "맑음, 28°C")
- **메뉴 추천 카드 3개**: 데스크톱은 가로 3열, 모바일은 세로로 쌓임
  - 메뉴 이름 (크게)
  - 추천 이유 (1~2줄)
- **"다시 추천받기" 버튼**: 누르면 AI에게 메뉴 추천을 다시 요청
- **로딩 상태**: 날씨 조회 중 / 메뉴 추천 생성 중 스켈레톤 또는 스피너 표시
- **에러 상태**: 날씨 API 또는 Claude API 호출 실패 시 안내 문구 + "다시 시도" 버튼

# 데이터 구조

v1은 회원/투표 기능이 없어서 기본적으로는 DB 저장 없이 동작 가능합니다 (매번 날씨 조회 → AI 추천 → 화면 표시).
다만 "있으면 좋은 것"에 있는 캐싱/히스토리를 구현할 경우 아래 테이블을 사용합니다.

## lunch_recommendations 테이블 (Supabase, 캐싱/히스토리용 — Nice to Have)
| 컬럼명 | 타입 | 설명 |
|---|---|---|
| id | uuid | 고유 식별자 (자동 생성) |
| weather_condition | text | 날씨 상태 (예: 맑음, 비, 눈) |
| temperature | numeric | 기온(°C) |
| feels_like | numeric | 체감온도(°C) |
| menus | jsonb | 추천 메뉴 3개 배열 (메뉴명 + 추천 이유) |
| created_at | timestamp | 생성 시각 (기본값: now()) |

# 외부 API 연동

- **날씨 API**: 서울 삼성동 좌표(위도 약 37.5087, 경도 약 127.0632) 기준으로 현재 날씨를 조회
  - 추천: OpenWeatherMap (무료 티어로 충분하고 연동이 간단함). 다른 API를 원하면 알려주세요.
  - 서버(Server Action)에서만 호출해서 API 키가 브라우저에 노출되지 않도록 함
- **Claude API (Anthropic)**: 날씨 정보(날씨 상태, 기온, 체감온도)를 프롬프트에 담아 전달하고,
  점심 메뉴 3개 + 각 메뉴의 추천 이유를 JSON 형태로 응답받음
  - 역시 서버(Server Action)에서만 호출

# 환경 변수 (추가 필요)
- `WEATHER_API_KEY` — 날씨 API 키 (예: OpenWeatherMap API Key)
- `ANTHROPIC_API_KEY` — Claude API 키 (서버 전용, `NEXT_PUBLIC_` 접두사 절대 사용 금지)

`.env.local`에 추가하고, `.env.example`에도 예시 값을 함께 업데이트합니다.
배포 시 Vercel 대시보드 → Settings → Environment Variables에도 동일하게 등록해야 합니다.

# 디자인 방향
- 식욕을 돋우는 따뜻한 포인트 컬러 (주황/노랑 계열) + 날씨 카드는 하늘색/그레이 계열로 대비
